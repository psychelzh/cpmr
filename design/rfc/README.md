# cpmr Architecture RFCs

This folder tracks architecture decisions and historical proposals for the
native-first `cpmr` redesign.

The `Status` line at the top of each RFC is authoritative. Some RFCs remain
useful implementation records; others are kept only to document decisions that
were later deferred or superseded.

## Index

1. [RFC 0001: CPM Core Engine Stabilization](./0001-core-engine.md)
   Status: Accepted
2. [RFC 0002: Tidymodels Adapter for CPM](./0002-tidymodels-adapter.md)
   Status: Deferred
3. [RFC 0003: Migration and Deprecation Strategy](./0003-migration-and-deprecation.md)
   Status: Superseded

## Process

1. Open a PR that updates one or more RFC files.
2. Discuss scope, compatibility impact, and rollout plan.
3. Update RFC status when direction changes, even if a historical file is
   retained for context.
